# Contributing to Awesome Vibe Coding

Thanks for wanting to add to this list. Before you open a PR, read this — it'll save us both time.

## The bar

This isn't a comprehensive list of every AI coding tool that exists. It's a **curated list with a clear point of view.**

Every entry must answer YES to all three:

1. **Would it help a non-technical founder ship a real product?**
2. **Has it shipped something real?** (You used it. A friend you trust used it. Not "I saw a demo.")
3. **Does it earn its spot on a small list, vs. just being good?** Quality > completeness.

If a tool is excellent but the audience is "senior engineers who already know what they're doing," it doesn't belong here.

If a tool is trendy on X but doesn't actually help solo non-technical builders, it doesn't belong here.

If a tool is genuinely useful to the audience but you've never used it, please ask in an issue first before PR-ing.

## What we accept

- ✅ Tools, services, and products that meet the bar above
- ✅ Real public repos from non-technical founders who shipped (in the "Real Builds" section)
- ✅ Books, podcasts, courses, and learning resources that actually move builders forward
- ✅ Communities where vibe coders genuinely hang out and learn

## What we don't accept

- ❌ Anything you're being paid to promote (without disclosure)
- ❌ Tools you haven't used personally
- ❌ Self-promotion with no track record
- ❌ Generic "10 best AI tools" content
- ❌ Closed-source tools with no free tier (unless they're genuinely best-in-class)
- ❌ Anything that requires a sales call to evaluate

## How to PR

1. Fork this repo
2. Add your entry in the right section, alphabetically when reasonable
3. Write the entry in this format:
   - **[Tool Name](https://link.com)** — One sentence on what it is. One sentence on why it's good for vibe coders. *Italicized note if it's a personal recommendation.*
4. Keep entries to 1–2 sentences max. Brevity is part of the value.
5. Commit message: `Add [Tool Name] to [Section]`
6. Open a PR with:
   - Why this tool meets the bar
   - Whether you've personally used it (and what you shipped with it, if anything)
   - Any conflicts of interest

## Suggesting a new section

If you think there's a category we're missing, open an issue first. Don't just PR a new section. The structure of this list is intentional and we want to keep it tight.

## Removing entries

If a tool dies, gets acquired and ruined, or stops being useful for the audience — open an issue or PR to remove it. Lists rot. We need help keeping it fresh.

## Code of Conduct

Be cool. This is a list maintained by people who actually ship; we have no time for drama. Disagreements are fine. Disrespect isn't.
